SECTION 1 - COMPLEXITY ANALYSIS

Q1. What Is Big O Notation?

ROI: ★★★★★ Frequency: Extremely High

Big O describes how an algorithm's running time (or memory) grows as the input size grows, ignoring constants and lower-order terms. It expresses the worst-case upper bound, which is what interviewers usually care about.

The point is to compare algorithms independent of hardware. An O(n) algorithm beats an O(n²) one for large inputs no matter whose laptop it runs on, because we care about the shape of the growth, not the exact milliseconds.

Interview Gold Point

Always state both time and space complexity. A common follow-up after you give a fast solution is "can you do it without extra memory?" — which is really asking you to trade O(n) space for an in-place O(1)-space approach.

Q2. How Do You Analyse the Complexity of a Loop?

ROI: ★★★★★ Frequency: Very High

Count how many times the innermost work runs as a function of n:

  • A single loop over n elements → O(n)
  • A loop nested inside another, both over n → O(n²)
  • A loop that halves the range each step (like binary search) → O(log n)
  • Two separate (not nested) loops → O(n) + O(n) = O(n), not O(n²)
// O(n) - one pass
for (int i = 0; i < n; i++) { sum += a[i]; }

// O(n^2) - nested
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        if (a[i] == a[j]) count++;

Common Mistake

Sequential loops are added, not multiplied. Two back-to-back O(n) loops are still O(n) overall — only nested loops multiply. Getting this wrong is a frequent way candidates over- or under-state complexity.

Q3. What Are the Common Complexity Classes?

ROI: ★★★★☆ Frequency: High

Complexity Name Typical Example
O(1) Constant HashMap get, array index access
O(log n) Logarithmic Binary search, balanced BST lookup
O(n) Linear Single loop, linear scan
O(n log n) Linearithmic Efficient sorting (merge, quick avg)
O(n²) Quadratic Nested loops, bubble sort
O(2ⁿ) Exponential Naive recursive Fibonacci, subsets

Interview Favorite

Know that O(n log n) is the best achievable for general comparison-based sorting. If you claim to sort faster than that, the interviewer will expect a non-comparison sort (counting/radix) with specific constraints on the input.

SECTION 2 - ARRAYS & STRINGS

Q4. Two Sum — Find Two Numbers That Add to a Target

ROI: ★★★★★ Frequency: Extremely High

The naive solution checks every pair in O(n²). The optimal solution uses a HashMap to remember numbers already seen, so each element is checked in O(1): for each number, ask "have I seen target - current before?" 

int[] twoSum(int[] nums, int target) {
    Map<Integer,Integer> seen = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (seen.containsKey(need))
            return new int[]{ seen.get(need), i };
        seen.put(nums[i], i);
    }
    return new int[]{-1,-1};
}
// Time: O(n)   Space: O(n)

Interview Gold Point

This is the canonical "trade space for time" answer. The HashMap-frequency pattern (count or remember what you have seen) solves a huge family of problems: first non-repeating character, anagrams, subarray sums, and more.

Q5. Explain the Two-Pointer Technique

ROI: ★★★★★ Frequency: Very High

Two pointers walk through a (usually sorted) array from opposite ends or at different speeds, replacing a nested loop with a single pass. Classic use: find a pair summing to a target in a sorted array. 

boolean hasPair(int[] sorted, int target) {
    int lo = 0, hi = sorted.length - 1;
    while (lo < hi) {
        int sum = sorted[lo] + sorted[hi];
        if (sum == target) return true;
        if (sum < target) lo++;   // need bigger
        else hi--;                // need smaller
    }
    return false;
}
// Time: O(n)   Space: O(1)

Interview Favorite

Two pointers also power: reversing in place, removing duplicates from a sorted array, merging two sorted arrays, and the "fast & slow" pointer for cycle detection (next question).

Q6. Sliding Window — Longest Substring Without Repeating Characters

ROI: ★★★★☆ Frequency: High

A sliding window maintains a contiguous range [left, right] and expands or shrinks it instead of recomputing from scratch. It turns many O(n²) substring/subarray problems into O(n). 

int longestUnique(String s) {
    Set<Character> window = new HashSet<>();
    int left = 0, best = 0;
    for (int right = 0; right < s.length(); right++) {
        while (window.contains(s.charAt(right)))
            window.remove(s.charAt(left++));   // shrink
        window.add(s.charAt(right));           // expand
        best = Math.max(best, right - left + 1);
    }
    return best;
}
// Time: O(n)   Space: O(k) distinct chars

Interview Gold Point

Recognise the trigger words: "longest / shortest / maximum / minimum contiguous subarray or substring". That almost always means a sliding window.

SECTION 3 - LINKED LISTS, STACKS & QUEUES

Q7. Reverse a Singly Linked List

ROI: ★★★★★ Frequency: Very High

Walk the list once, re-pointing each node's next to the previous node. Track three references: previous, current, and the next node (saved before you overwrite the link). 

Node reverse(Node head) {
    Node prev = null, curr = head;
    while (curr != null) {
        Node next = curr.next;  // save
        curr.next = prev;       // reverse link
        prev = curr;            // advance
        curr = next;
    }
    return prev;                // new head
}
// Time: O(n)   Space: O(1)

Common Mistake

Forgetting to save curr.next before reassigning it — you lose the rest of the list. Saying the three-pointer dance out loud before coding prevents this.

Q8. Detect a Cycle in a Linked List

ROI: ★★★★☆ Frequency: High

Use Floyd's fast & slow pointers: slow moves one step, fast moves two. If they ever meet, there is a cycle; if fast reaches null, there is none. This is O(1) space, unlike the HashSet approach which is O(n). 

boolean hasCycle(Node head) {
    Node slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}
// Time: O(n)   Space: O(1)

Interview Gold Point

The follow-up is often "find where the cycle begins". The trick: after they meet, reset one pointer to the head and advance both one step at a time — they meet at the cycle's start.

Q9. Valid Parentheses — Why a Stack?

ROI: ★★★★☆ Frequency: High

Matching brackets is a "last opened, first closed" problem — exactly what a stack models. Push every opening bracket; on a closing bracket, the top of the stack must be its matching opener. 

boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    Map<Character,Character> pairs = Map.of(')','(', ']','[', '}','{');
    for (char c : s.toCharArray()) {
        if (pairs.containsValue(c)) stack.push(c);
        else if (stack.isEmpty() || stack.pop() != pairs.get(c))
            return false;
    }
    return stack.isEmpty();
}
// Time: O(n)   Space: O(n)

Interview Favorite

Prefer ArrayDeque over the legacy Stack class in Java — Stack extends Vector and is synchronised (slower), while ArrayDeque is the modern, faster choice for stack/queue behaviour.

SECTION 4 - SEARCHING, SORTING & TREES

Q10. Implement Binary Search

ROI: ★★★★★ Frequency: Very High

Binary search finds a target in a sorted array in O(log n) by repeatedly halving the search range. The classic bug is integer overflow when computing the midpoint — use lo + (hi - lo) / 2. 

int binarySearch(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] == target) return mid;
        if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
// Time: O(log n)   Space: O(1)

Interview Gold Point

Mention the two classic pitfalls: the (lo + hi) overflow, and getting the loop boundary wrong (<= vs <). Java's Arrays.binarySearch exists, but interviewers usually want the hand-written version.

Q11. Explain Tree Traversals (BFS and DFS)

ROI: ★★★★☆ Frequency: High

DFS (Depth-First) goes as deep as possible before backtracking; implemented with recursion or an explicit stack. It has three orders: pre-order (node, left, right), in-order (left, node, right — gives sorted output for a BST), and post-order (left, right, node).

BFS (Breadth-First / level-order) visits the tree level by level using a queue. Use it when you care about distance from the root or "shortest path in an unweighted graph". 

// BFS level-order
void bfs(Node root) {
    if (root == null) return;
    Queue<Node> q = new LinkedList<>();
    q.add(root);
    while (!q.isEmpty()) {
        Node n = q.poll();
        visit(n);
        if (n.left != null)  q.add(n.left);
        if (n.right != null) q.add(n.right);
    }
}

Interview Favorite

Remember: in-order traversal of a BST yields sorted values. This single fact answers many BST questions ("validate a BST", "kth smallest element").

Q12. Sorting Algorithms — Which One and Why?

ROI: ★★★★☆ Frequency: Medium-High

Algorithm Average Worst Stable Notes
Quick Sort O(n log n) O(n²) No Fast in practice, in-place
Merge Sort O(n log n) O(n log n) Yes Stable, needs O(n) space
Heap Sort O(n log n) O(n log n) No In-place, no recursion
Bubble/Insertion O(n²) O(n²) Yes Only for tiny/nearly-sorted

Interview Gold Point

Java's Arrays.sort uses a dual-pivot Quicksort for primitives (raw speed, stability irrelevant) and Timsort for objects (a stable merge/insertion hybrid). Knowing why they differ — stability matters for objects, not primitives — is a strong senior signal.

SECTION 5 - RECURSION & DYNAMIC PROGRAMMING

Q13. What Makes a Correct Recursive Solution?

ROI: ★★★★☆ Frequency: Medium-High

Every recursion needs two things: a base case that stops the recursion, and a recursive case that makes progress toward the base case. Miss the base case (or never approach it) and you get a StackOverflowError.

Mentally, trust that the recursive call returns the correct answer for the smaller problem, then combine. This "leap of faith" is what makes recursion click.

Interview Gold Point

Deep recursion risks a stack overflow in Java because there is no automatic tail-call optimisation. For very deep inputs, convert to an iterative solution with an explicit stack.

Q14. Fibonacci — Recursion vs Memoization (Dynamic Programming)

ROI: ★★★★☆ Frequency: High

Naive recursive Fibonacci is O(2ⁿ) because it recomputes the same subproblems exponentially. Dynamic programming fixes this by storing results: memoization (top-down cache) or tabulation (bottom-up array), turning it into O(n). 

// Bottom-up DP - O(n) time, O(1) space
long fib(int n) {
    if (n < 2) return n;
    long a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        long c = a + b;
        a = b; b = c;
    }
    return b;
}

Interview Gold Point

The DP signal is "overlapping subproblems + optimal substructure". When you notice the same inputs being recomputed, reach for memoization. Mention you can often drop the table to O(1) space when each step only needs the previous one or two results.

Q15. Array vs LinkedList vs HashMap — How Do You Choose?

ROI: ★★★★★ Frequency: Very High

Operation Array LinkedList HashMap
Access by index/key O(1) O(n) O(1) avg
Search by value O(n) O(n) O(1) avg
Insert/delete at front O(n) O(1) O(1) avg
Memory overhead Low High (pointers) Medium

Rule of thumb: need fast lookup by key or de-duplication → HashMap/HashSet. Need indexed access and iteration → array/ArrayList. Need constant-time insert/remove at the ends → LinkedList or ArrayDeque.

Interview Gold Point

The most common real-world answer is "use a HashMap" — many brute-force O(n²) solutions collapse to O(n) the moment you introduce a map to remember what you have already seen.

QUICK REVISION SHEET

  • Big O = growth rate, worst case; always state time AND space
  • Nested loops multiply; sequential loops add
  • HashMap pattern: remember what you've seen → O(n²) becomes O(n)
  • Two pointers: sorted arrays, in-place, fast/slow cycle detection
  • Sliding window: longest/shortest contiguous subarray or substring
  • Stack = matching/LIFO; Queue = BFS/level-order
  • Binary search needs sorted input; watch midpoint overflow
  • In-order BST traversal → sorted output
  • DP = overlapping subproblems; memoize or tabulate
  • Use ArrayDeque, not the legacy Stack class
  • Talk through the approach and edge cases BEFORE coding
